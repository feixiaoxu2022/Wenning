# 工具选择策略设计

## 核心原则（基于"Just Right"平衡）

✅ **DO**: 提供清晰的工具特点和适用场景
❌ **DON'T**: 强制"必须使用XX工具"的硬性规则
✅ **DO**: 给出选择指导原则
❌ **DON'T**: 列举详尽的if-else条件

---

## 1. TTS工具选择指南

### tts_minimax
**特点**: 中文优化、情感表达丰富
**适用**: 中文内容、需要情感色彩的语音（happy/sad/angry等）
**示例场景**: 有声小说、情感化客服回复

### tts_google
**特点**: 多语言支持、质量稳定、语音库丰富
**适用**: 多语言需求、标准商业应用
**示例场景**: 多语言教学、产品介绍

### tts_azure
**特点**: 企业级质量、精细参数调节
**适用**: 对音质要求高的专业场景
**示例场景**: 企业培训、专业播报

### tts_local
**特点**: 无需API、快速响应
**适用**: 快速原型、测试场景
**示例场景**: 开发调试、简单提醒

## 2. 图像生成工具选择指南

### image_generation_minimax
**特点**:
- 使用宽高比（16:9、1:1等）
- 支持prompt_optimizer自动优化描述
- 艺术风格生成能力强

**适用**:
- 创意设计、艺术创作
- 不确定具体尺寸，只知道宽高比
- 需要AI优化prompt描述

**示例场景**:
- 社交媒体封面（16:9）
- 海报设计
- 创意插图

### text_to_image_minimax
**特点**:
- 精确的width×height控制
- 支持quality参数（standard/high）
- 适合标准化输出

**适用**:
- 明确尺寸需求（如1920×1080）
- 需要批量生成统一规格图片
- 对质量有明确要求

**示例场景**:
- 网站banner（固定尺寸）
- 产品展示图（统一规格）
- 打印素材（精确尺寸）

### code_executor（图像生成）
**特点**:
- Python代码生成（PIL、matplotlib）
- 完全可控的绘制逻辑
- 适合技术图形和数据可视化

**适用**:
- 数据图表、统计图
- 技术示意图、流程图
- 需要精确控制每个元素

**示例场景**:
- 数据分析报告图表
- 算法流程图
- 像素级精确设计

**选择建议**: 艺术创作 → MiniMax API；数据可视化/技术图形 → code_executor

## 3. 视频生成工具选择指南

### video_generation_minimax
**特点**:
- AI生成自然场景视频
- 6秒短视频
- 支持720P/1080P分辨率

**适用**:
- 自然场景、人物动作
- 创意短视频
- AI生成内容

**示例场景**:
- 产品宣传片段
- 自然风景视频
- 创意视觉效果

### code_executor + matplotlib.animation
**特点**:
- 数据动画、图表演变
- 完全可控的动画逻辑
- 适合技术演示

**适用**:
- 数据变化动画
- 算法可视化
- 教学演示

**示例场景**:
- 股票走势动画
- 排序算法演示
- 科学数据可视化

### code_executor + moviepy
**特点**:
- 视频剪辑、合成
- 文字叠加、转场效果
- 音视频同步

**适用**:
- 视频编辑任务
- 多素材合成
- 字幕、特效添加

**示例场景**:
- 解说视频制作
- 宣传片剪辑
- 教程视频合成

**选择建议**: AI生成内容 → MiniMax API；数据动画 → matplotlib；视频编辑 → moviepy

## System Prompt集成建议

### ❌ 不好的方式（Too Specific）
```
当用户要求生成图片时，你必须按照以下步骤判断：
1. 如果用户说了16:9，则必须使用image_generation_minimax
2. 如果用户说了具体像素，则必须使用text_to_image_minimax
3. 如果是数据图表，则必须使用code_executor
...
```

### ✅ 好的方式（Just Right）
```
## 图像生成工具

你有多个图像生成工具可用：
- `image_generation_minimax`: 艺术创作，支持宽高比和prompt优化
- `text_to_image_minimax`: 精确尺寸控制，适合标准化输出
- `code_executor`: 数据可视化和技术图形

选择原则：艺术创意类任务优先考虑MiniMax API；数据图表和技术图形使用code_executor。
```

### ❌ 不好的方式（Too Vague）
```
根据需求选择合适的工具生成图片。
```

---

## 总结

**核心思想**:
- 描述工具特点和适用场景，而非强制规则
- 提供选择指导原则，而非详尽条件列表
- 给模型判断空间，而非限制其灵活性
